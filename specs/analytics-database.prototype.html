Data Flow

Events flow from client widgets through the server ingestion endpoint into a D1 database, then get rolled up by a daily cron into summary tables that power the admin dashboard.

Write Path (hot)

Real-time
1. Client POST batch of events
2. Server applies per-type sampling
3. Batch INSERT surviving events into events
4. Atomic increment counters for funnel events

Read Path (dashboard)

Aggregated
Today: Read from counters table
Historical: Query daily_summaries
Drill-down: Query raw events (30d window)
Ad-hoc: json_extract() on body

Sampling Strategy

Server-side sampling applied per event type. Key funnel events are tracked at 100%. High-volume events are sampled and extrapolated using the stored sample_rate.
